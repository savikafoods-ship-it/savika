SAVIKA - PRODUCT REQUIREMENTS DOCUMENT (PRD)
================================================================================

1. PRODUCT VISION
Savika is a premium D2C brand offering 100% pure, authentic Indian spices and 
artisan blends. Our goal is to provide a high-quality, trustworthy e-commerce 
experience from discovery to delivery.

2. DETAILED SITEMAP (STOREFRONT)
* Landing Page (/): Hero section, category discovery, featured products, best 
  sellers, and social proof.
* Shop (/shop): Full product catalog with filters and search.
* Category Page (/category/[id]): Curated product lists by spice category.
* Product Detail (/product/[slug]): Image carousel, dynamic pricing by variant, 
  detailed descriptions, and related products.
* Shopping Cart (/cart): Real-time cart management with local persistence.
* Checkout (/checkout): Multi-step/One-page secure checkout flow.
* User Account (/account): Profile management and basic user settings.
* My Orders (/orders): Historical order overview and status tracking.
* Wishlist (/wishlist): Save products for later viewing.
* Brand Content:
    - Our Story (/our-story)
    - Why Savika (/why-savika)
    - Contact Us (/contact)
* Legal/Policies:
    - Privacy Policy (/privacy)
    - Refund Policy (/refund-policy)
    - Shipping Policy (/shipping-policy)
    - Terms & Conditions (/terms)

3. DETAILED SITEMAP (ADMIN PANEL)
Designed for efficient site management with a responsive mobile interface.
* Dashboard (/admin/dashboard): KPI overview (sales, orders, visitors).
* Orders (/admin/orders): Manage order fulfillment and view details.
* Products (/admin/products): CRUD operations, inventory and price variants.
* Categories (/admin/categories): Organize product taxonomy.
* Customers (/admin/customers): View and manage user registration data.
* Coupons (/admin/coupons): Create and monitor promotional discount codes.
* Analytics (/admin/analytics): Data visualization for sales and user trends.
* Updates (/admin/updates): Manage dynamic site content (banners, announcements).
* Settings (/admin/settings): Global site configuration and store details.
* Admin Admins (/admin/admins): Role-based access control for team members.
* Admin Login (/admin/login): Secure portal for administrative access.

================================================================================
SAVIKA - TECHNICAL REQUIREMENTS DOCUMENT (TRD)
================================================================================

1. CORE STACK
* Framework: Next.js 15+ (App Router).
* Styling: Tailwind CSS (Primary for layouts and components).
* Icons: Font Awesome (Professional SVG icons for all UI elements).
* Database & Backend: Supabase (Auth, PostgreSQL DB, and Cloud Storage).
* State Management: Zustand (Storefront cart and Admin UI states).

2. ADMIN PANEL ARCHITECTURE
* Mobile First: The entire admin dashboard is built with a mobile-responsive 
  grid system. Navigation adapts from a persistent sidebar on desktop to a 
  collapsible menu (hamburger) on mobile.
* Interactivity: Dynamic data tables, filterable lists, and real-time form 
  validation.

3. SECURITY & INTEGRATION
* RLS (Row Level Security): Enforced via Supabase to protect admin data.
* Admin Authentication: Dedicated secure login flow distinct from storefront.

================================================================================
SAVIKA - DESIGN GUIDELINES
================================================================================

1. GENERAL DESIGN PRINCIPLES
* Authenticity: Earthy tones reflecting natural spices.
* Premium Feel: Clean spacing, subtle shadows, and high-quality iconography.
* Mobile-Responsive: 100% of the site features must be usable on mobile devices, 
  particularly the Admin Panel.

2. ICONOGRAPHY & BRANDING
* Library: Font Awesome. All icons (nav, actions, indicators) must be sourced 
  from Font Awesome to ensure consistency.
* Primary Colors: 
    - Gold/Amber (#C47F17) for CTAs and highlights.
    - Stone/Cream (#F5F0E8) for background foundations.

3. LAYOUT SPECIFICATIONS
* Typography: Poppins (Sans-serif) for all digital interfaces.
* Components: Standardized card system for products and admin stats.
* Admin Sidebar: Dark-themed navigational sidebar (as per design reference) 
  with gold-active states for better visual hierarchy.

4. RESPONSIVENESS (MOBILE-FIRST)
* Grid System: Flexbox and Grid based on Tailwind breakpoints (sm, md, lg, xl).
* Touch Optimization: Large touch targets for buttons and interactive menus.
